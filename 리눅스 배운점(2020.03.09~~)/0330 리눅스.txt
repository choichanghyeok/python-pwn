1. 텔넷 ( 보안상의 문제때문에 잘 사용하지않음)	

취약점 >> IP spoofing 과 DNS poisoning

텟넬서버 설치하기

dpkg -s telnetd	<< 명령어를 입력해서 설치가되어있는지 확인

-s( 명령어로 상태를 알수있음)
d >> 서비스 프로그램


apt search telnetd 명령어를 입력해서 설치할수있는 버전이 있는지 확인



sudo apt install telnetd 를 입력해서 텔넷을 설치한다.


그리고 윈도우에 프로그램 및 기능의 윈도우 기능 켜기/끄기에 들어가서

텔넷 클라이언트를 체크하고 확인 눌러주면 된다.


포토포워딩을 하기위해서 돌아가서 접속한다.

ex)

텔넷은 23번 포트를 사용함

가상머신을 선택해 설정을 누름 >> 네트워크 클릭 >> 고급 클릭>> 포토포워딩 클릭 >> 프로토콜 : TCP , 호스트 IP: , 포트 : 23 , 게스트 IP : 자신의 IP ,게스트포트: 23



우분투 리눅스 IP와 윈도우 IP 가 필요함 리눅스 = ifconfig입력  윈도우 = ipconfig

어차피 자기자신의 컴퓨터라면 telnet localhost 라고 적어도 된다.


그리고 cmd에서

telnet IP주소 입력해서 리눅스 계정과 패스워드를 입력해서 로그인

(구글에 공유기에게 포트포워딩 검색해서 공부하기)


2. ssh

윈도우 도스창에서 ipconfig 에서 DNS 서버 IP를 볼수있다.


네임서버가 하는 역할은 물어봐 주는 역할이다. 

이 IP가 무엇인지 같은 정보를 물어봐주는 역할(즉 중간자역할)

단 이 DNP poisoning 이 취약한 이유는 

애가 물어보지도않았는데 응답성 패킷을 보내서 거짓정보를 보내게되면 네임서버는 학습(기억)을 해버린다.

이상태에서 또다른 사용자가 다시 물어보게되면 잘못된정보를 알려주게된다.


즉 IP spoofing 은 중간자 공격으로 보안상 위험이있고

DNS poisoning 은 위조 즉 홈페이지 위조같은걸로 사람을 속여서 아이디와 비밀번호를 입력하게한다.



텔넷 보다는 안전한 ssh를 구현한 방식

>> 첫 연걸에서 암호를 푸는 키를 나누어 갖고 그것으로 모든 메세지는 암호로 보낸다.

이로써 텔넷보다는 ssh가 보안상으로 확실하므로 ssh를 사용한다.
