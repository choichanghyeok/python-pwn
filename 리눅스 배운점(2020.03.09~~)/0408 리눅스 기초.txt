@@@@@ 서비스와 소켓@@@@@

#서비스 >> 시스템과 독자적으로 구동되어 제공하는 프로세스를 말한다. 옐 웹서버, DB서버, FTP서버 등이 있다.
실행및 종료는	"systemctl start/stop/restart 서비스이름"   으로 사용

서비스 실행 스크립트 파일은 /lib/systemd/system/ 디렉터리에 "서비스이름.service" 라는 이름으로 확인이 가능
ex) Cron 서비스는 cron.service 라는 이름의 파일로 존재

또 부팅과 동시에 서비스의 자동 실행여부를 지정이 가능함, 터미널에서 systemctl list-unit-files 명령을 실행, 
현재사용(enabled), 사용 안함(disabled) 확인가능 또 rcconf 명령으로 설정가능



#소켓 >> 서비스는 항상 가동, 소켓은 외부에서 특정 서비스를 요청 할경우에 systemd가 구동, 요청이 끝나면 소켓 종료

그래서 소켓으로 설정된 서비스를 요청할시 처음연결되는 시간이 서비스들과 비교했을시 약간 더 걸릴수있다.

이유 >> systemd가 서비스를 새로 구동하는데 시간이 소요됨, 이와같이 대표적인 예로는 텔넷이 있음

소켓과 관련된 스크립트 파일은 /lib/systemd/system/ 디렉터리에 소켓이름.socket 이라는 이름으로 존재

GRUB 부트로더

Grub 부트로더의 특징 >> 부트 정보를 사용자가 임의로 변경해 부팅할 수가 있음, 즉 부트정보가 올바르지 않더라도 수정하여 부팅할수있다.

또 다른 여러가지 운영체제와 멀티 부팅이 가능 , 대화형 설정을 제공 그래서 커널의 경로와 파일 이름만 알면 부팅이 가능

GRUB2의 장점

>> 셀 스크립트를 지원함으로써 조건식과 함수를 사용가능 , 동적모듈을 로드 가능 , iso 이미지를 이용해서 바로 부팅이 가능


GRUB2 설정 방법

/boot/grub/grub.cfg 설정파일 ( 직접 변경하면 안됨)
/etc/default/grub 파일과 /etc/grub.d/ 디렉터리의 파일을 수정 후 " grub-mkconfig "  명령어를 실행해 설정

RAID 정의 및 개념

RAID 정의

>> RAID는 여러개의 디스크를 하나의 디스크처럼 사용, 	비용절감 + 신뢰성 향상 + 성능 향상의 효과를 냄

하드웨어 RAID

>> 하드웨어 제조업체에서 여러 개의 하드디스크를 가지고 장비를 만들어 그자체를 공급 , 좀더 안정적이지만 상당히 고가임

소프트웨어 RAID

>> 고가의 하드웨어 RAID의 대안 , 운영체제에서 지원하는 방식, 저렴한 비용으로 좀더 안전한 데이터의 저장이 가능


##linear RAID , RAID0

linear RAID 개요

> 최소 2개의 하드디스크 필요, 2개이상의 하드디스크를 1개의 볼륨으로 사용 , 앞디스크 부터 차례로 저장 100%의 공간의 효율성(비용이 저렴)

RAID0 개요

> 최소 2개의 하드디스크 필요, 모든 디스크에 동시에 저장, 00%의 공간의 효율성(비용이 저렴), 신뢰성이 낮음
빠른성능을 요구하되 전부잃어버려도 큰문제가 되지않는 자료가 적당

RAID1 개요

> 미러링 이라부름, 데이터 저장에 두배의 용량이 필요, 결함허용을 제공 =신뢰성이 높음, 두배의 저장공간 = 비용이 두배 = 효율이 나쁨
저장속도(성능)은 변함없음, 중요한데이터를 저장하기에는 적합함.


비교

구분				RAID0				RAID1

성능(속도)			뛰어남				변화 없음

데이터 안전성(결함허용)		보장 못함(결함 허용x)		보장함(결함 허용o)

공간 효율성			좋음				나쁨




RAID5 개요

>> RAID1의 데이터의 안전성 + RAID처럼 공간 효율성, 최소한 3개 이상의 하드디스크 , 오류가 발생할떄는 패리티를 이용해서
데이터를 복구함.

RAID5 특징

>> 어느정도의 결함허용을 해주면서 저장공간의 효율도 좋고 디스크의 개수 -1의 공간을 사용 또 디스크 2개가 고장나면 복구못함



RAID6 특징

>> RAID 방식은 RAID5 방식이 개선된 것 공간효율은 RAID5 보다 약간 떨어짐, 그렇지만 2개의 디스크가 동시에 고장나도 데이터에는
이상이 없도록 하는 방식

RAID6의 경우 최소 4개의 디스크가 필요 , 공간효율은 RAID5보다 조금 떨어지지만 데이터에관한 신뢰도는 좀더 높음
성능은 RAID5 에 비해 조금 떨어진다.
